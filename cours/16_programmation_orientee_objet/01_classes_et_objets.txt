1. PARADIGME ORIENTÉ OBJET

La Programmation Orientée Objet (POO) est un paradigme de programmation qui organise le code autour d'objets plutôt que de fonctions et de procédures.

Avantages :
- Modélisation proche du monde réel
- Réutilisabilité du code (héritage, composition)
- Maintenabilité (encapsulation, cohésion)
- Extensibilité (polymorphisme, interfaces)

Les 4 piliers de la POO :
1. Encapsulation : cacher les détails internes, exposer une interface publique
2. Héritage : créer de nouvelles classes à partir de classes existantes
3. Polymorphisme : une interface, plusieurs comportements
4. Abstraction : simplifier la complexité en exposant l'essentiel

Langages OOP : Java, Python, C++, C#, PHP, Ruby, Swift, Kotlin

2. CLASSES ET OBJETS

Une classe est un moule, un plan de construction. Un objet est une instance de classe.

Exemple en Python :
class Etudiant:
    """Représente un étudiant de BTS GL."""

    # Attribut de classe (partagé par toutes les instances)
    etablissement = "ISTA Yaoundé"

    def __init__(self, nom: str, prenom: str, matricule: str):
        """Constructeur : initialise les attributs d'instance."""
        self.nom = nom                  # attribut public
        self.prenom = prenom            # attribut public
        self._matricule = matricule     # attribut protégé (convention)
        self.__moyenne = 0.0            # attribut privé (name mangling)

    def get_nom_complet(self) -> str:
        return f"{self.prenom} {self.nom}"

    def set_moyenne(self, moy: float) -> None:
        if 0.0 <= moy <= 20.0:
            self.__moyenne = moy
        else:
            raise ValueError("La moyenne doit être entre 0 et 20")

    def get_moyenne(self) -> float:
        return self.__moyenne

    def __str__(self) -> str:
        """Représentation lisible (str())."""
        return f"Étudiant({self.get_nom_complet()}, moy={self.__moyenne})"

    def __repr__(self) -> str:
        """Représentation officielle (repr())."""
        return f"Etudiant('{self.nom}', '{self.prenom}', '{self._matricule}')"

# Utilisation
e1 = Etudiant("DJOMO", "Paul", "BGL2024001")
e1.set_moyenne(14.5)
print(e1)             # Étudiant(Paul DJOMO, moy=14.5)
print(e1.get_nom_complet())  # Paul DJOMO
print(Etudiant.etablissement)  # ISTA Yaoundé

Exemple en Java :
public class Etudiant {
    private String nom;
    private String prenom;
    private double moyenne;

    public Etudiant(String nom, String prenom) {
        this.nom = nom;
        this.prenom = prenom;
        this.moyenne = 0.0;
    }

    public String getNom() { return nom; }
    public String getPrenom() { return prenom; }
    public double getMoyenne() { return moyenne; }

    public void setMoyenne(double moy) {
        if (moy >= 0 && moy <= 20)
            this.moyenne = moy;
        else
            throw new IllegalArgumentException("Moyenne invalide");
    }

    @Override
    public String toString() {
        return String.format("Etudiant[%s %s, moy=%.1f]", prenom, nom, moyenne);
    }
}

3. ENCAPSULATION

Définition : regrouper les données (attributs) et les comportements (méthodes) au sein d'une même unité (classe), et contrôler l'accès depuis l'extérieur.

Niveaux d'accès :
- public (+) : accessible partout
- private (-) : accessible seulement depuis la classe elle-même
- protected (#) : accessible depuis la classe et ses sous-classes
- package (Java, sans modificateur) : accessible dans le même package

Getters et Setters (accesseurs et mutateurs) :
Permettent d'accéder aux attributs privés tout en validant les données.

Propriétés Python (@property) :
class Produit:
    def __init__(self, prix: float):
        self._prix = prix

    @property
    def prix(self) -> float:
        return self._prix

    @prix.setter
    def prix(self, valeur: float) -> None:
        if valeur < 0:
            raise ValueError("Prix négatif impossible")
        self._prix = valeur

p = Produit(5000)
print(p.prix)         # 5000 (appelle le getter)
p.prix = 6000         # appelle le setter

4. MÉTHODES SPÉCIALES (MAGIC METHODS EN PYTHON)

__init__ : constructeur
__del__ : destructeur (appelé quand l'objet est supprimé)
__str__ : conversion en chaîne (str(), print())
__repr__ : représentation officielle (repr())
__len__ : longueur (len())
__eq__ : égalité (==)
__lt__, __gt__, __le__, __ge__ : comparaisons
__add__, __sub__, __mul__ : opérateurs arithmétiques
__getitem__, __setitem__ : indexation (obj[i])
__iter__, __next__ : itération (for x in obj)
__contains__ : opérateur in
__enter__, __exit__ : gestionnaire de contexte (with)

Exemple :
class Vecteur2D:
    def __init__(self, x, y):
        self.x, self.y = x, y

    def __add__(self, autre):
        return Vecteur2D(self.x + autre.x, self.y + autre.y)

    def __abs__(self):
        return (self.x**2 + self.y**2) ** 0.5

    def __str__(self):
        return f"({self.x}, {self.y})"

v1 = Vecteur2D(3, 4)
v2 = Vecteur2D(1, 2)
print(v1 + v2)   # (4, 6)
print(abs(v1))   # 5.0
